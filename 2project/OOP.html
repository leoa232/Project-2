Encapsulation

Encapsulation is one of the major concepts in object oriented programming. It describes the idea of blocking access to methods and attributes in a class. This will keep the complicated details out of the sight of the users, and prevent data from accidently being changed. In Python, this is achieved by using private methods or attributes using underscore as prefix ie single “_” or double “__”. For example in the calculator code we created several static methods. In order to ensure these methods remained inaccessible, we needed to add the underscore. However, the singe underscore will not completely keep the method inaccessible. We need to have the double underscore. In order to get access to the double underscore attributes, we need to use getter and setter function to access it internally

Inheritance

Inheritance allows us to define a class that inherits all the methods and attributes from another class. Convention denotes the new class as child class, and the one that it inherits from is called parent class or superclass. If we refer back to the definition of class structure, we can see the structure for basic inheritance is class ClassName(superclass), which means the new class can access all the attributes and methods from the superclass. Inheritance builds a relationship between the child class and parent class, usually in a way that the parent class is a general type while the child class is a specific type. In the calculator program, our parent class was Calculation. Some of the child classes included addition, subtraction, multiplication, and addition.

Polymorphism

Polymorphism is another fundamental aspect of OOP, which means multiple forms. Polymorphism enables us to use a single interface with various underlying forms such as data types or classes. For example, we can have commonly named methods across classes or child classes. This ability of using single name with many forms acting differently in different situations greatly reduces our complexities. In the calculator program, we have several classes. Although the classes were different, they relied on similar methods. Polymorhism allows these classes to make use of the same methods.

Abstraction

In simple terms, abstraction hides the internal implementations of a process or method from the user. In this way, the user knows what they do but not how it is done. Abstraction is a bit different from the other pillars. Python does not grant abstract classes by default. Instead, Python comes with a module that fits the base for Abstract Base classes (ABC). A method becomes abstract when decorated with the keyword @abstractmethod. In the calculator program, there are many working components to achieve the calculator. Being able to keep the internal implementations from the users helps prevent the program from being comprimised.
